# Pre-Start Checklist

> Before writing a single line of code.

---

## On receiving a new task

- [ ] Read the entire brief **once, slowly** — including the last lines
- [ ] Read it a **second time**, this time underlining every deliverable and every constraint
- [ ] Write down in your own words: **what is the actual goal of this part?**
- [ ] List every explicit deliverable — nothing implied, only what is written
- [ ] Identify any **warnings, rules, or "do NOT"** sections — treat them as hard constraints
- [ ] If something is ambiguous, **ask before starting** — not halfway through

## On receiving a review with follow-ups

- [ ] Read the full review before touching any code
- [ ] Make a **numbered list** of every fix requested — including carryovers from previous reviews
- [ ] For each item: is it a quick fix, or does it require understanding something first?
- [ ] Keep that list open until every box is checked — it becomes your PREPUSH_CHECKLIST input

---

*If you haven't done this, you haven't started yet.*
